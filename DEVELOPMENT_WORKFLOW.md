# Робочий процес розробки

## GitFlow

- **main:** Код, готовий до розгортання.
- **develop:** Інтеграційна гілка для нових функцій.
- **feature/...:** Гілки для розробки нового функціоналу.
- **bugfix/...:** Гілки для виправлення помилок.
- **hotfix/...:** Гілки для термінових виправлень у продакшені.

## Code Review

- Усі pull request повинні бути перевірені щонайменше одним членом команди.
- Використовуйте шаблон pull request.

## CI/CD

- Автоматизовані збірки, тестування та розгортання запускаються при пуші в гілки `main` та `develop`.
