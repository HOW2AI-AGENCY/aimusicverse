# Рабочий процесс разработки

## GitFlow

- **main:** Код, готовый к развертыванию.
- **develop:** Интеграционная ветка для новых функций.
- **feature/...:** Ветки для разработки нового функционала.
- **bugfix/...:** Ветки для исправления ошибок.
- **hotfix/...:** Ветки для срочных исправлений в продакшене.

## Code Review

- Все pull request должны быть проверены как минимум одним членом команды.
- Используйте шаблон pull request.

## CI/CD

- Автоматизированные сборки, тестирование и развертывание запускаются при пуше в ветки `main` и `develop`.
