# ADR-001: Выбор технологического стека

- **Статус:** Принято
- **Дата:** 2025-11-30

## Контекст

Для проекта MusicVerse AI требовался современный, масштабируемый и быстроразвивающийся стек, который идеально подходит для создания Telegram Mini App с бэкендом как услугой (BaaS).

## Рассмотренные варианты

- **Вариант 1: Next.js + Vercel + Firebase.** Мощный стек, но более сложный в настройке для Telegram Mini App и менее гибкий в плане работы с базой данных.
- **Вариант 2: React + Vite + Supabase.** Легковесный и быстрый фронтенд с Vite. Supabase предоставляет PostgreSQL, аутентификацию и Edge Functions, что идеально сочетается с требованиями проекта.

## Решение

Выбран **Вариант 2 (React + Vite + Supabase)**. Эта комбинация обеспечивает быструю разработку, отличную производительность и идеальную интеграцию с Telegram через Edge Functions для безопасной аутентификации. PostgreSQL от Supabase предлагает гибкость для сложных запросов, необходимых для системы рекомендаций.

## Последствия

- **Положительные:** Быстрый старт, отличная производительность, безопасность на уровне BaaS, гибкость SQL.
- **Отрицательные:** Меньшее "коробочное" решение по сравнению с Vercel, требует самостоятельной настройки некоторых аспектов CI/CD.
